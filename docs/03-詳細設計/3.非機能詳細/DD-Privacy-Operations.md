# プライバシー/監査運用（詳細）

## 対応NFR
NFR-PRC-001〜008

## データ台帳（PRC-001）
- データ項目
  - アカウント情報/ノート本文/共有情報/監査ログ
- 機密区分
  - PII/業務データ/監査ログ
- 保存先
  - DB/オブジェクトストレージ/監査ストレージ
- アクセス権
  - 役割ベース、最小権限
- ログ方針/保持期間
  - 主要ログ30日、監査ログ365日

## マスキング/テストデータ（PRC-002）
- ログマスキング規約
  - email/氏名/電話番号はマスクまたはハッシュ化
- 非本番データ生成
  - 本番データのコピー禁止
  - 合成データ生成スクリプトを使用
- 障害解析向け
  - 追跡はtraceId/requestIdで実施（生PIIは保持しない）

## 保持/削除（PRC-003）
- 自動削除/匿名化ジョブ仕様
  - 退会から30日後に論理削除
  - 90日後に物理削除
- 退会と整合する削除シーケンス
  - ユーザ削除 → 共有解除 → 添付削除 → 監査ログ保持

## 監査証跡（PRC-004）
- 監査イベント一覧
  - 認証/権限変更/データ削除/設定変更
- 保管/検索/改ざん耐性の設定
  - WORM相当ストレージ
  - 365日保持

## データ主体要求（PRC-007）
- 受付→抽出→削除/訂正→完了 の手順
- 期限: 30日以内
- 対応履歴を監査ログに記録

## エビデンス提示（PRC-008）
- エビデンス索引
  - 設定証跡/テスト結果/訓練記録/ログ
- 5営業日以内提示の演習手順
  - 四半期に1回実施
