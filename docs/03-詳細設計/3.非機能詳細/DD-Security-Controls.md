# セキュリティ統制（詳細）

## 対応NFR
NFR-SEC-001〜012（特にIAM/暗号/監視/ベースライン/供給網）

## IAMロール定義
- Admin: 全権限、MFA必須
- Operator: 運用操作、変更権限は限定
- Developer: 読み取り+デプロイ権限（本番は承認必須）
- Service: サービス間通信のみ
- 最小権限ポリシー、例外は期限付き（最大7日）
- 監査補足: cdk-nagのAwsSolutionsChecksでIAM/暗号化/監査要件を検証

## cdk-nag抑制の運用
- 抑制には理由と適用範囲を必須化し、コードレビューで妥当性を確認
- 変更管理記録に抑制IDと理由を残す
- 四半期ごとに抑制一覧を監査し、不要な抑制は削除

## 暗号/鍵/証明書
- TLS設定
  - TLS1.2/1.3のみ許可
  - 旧式スイート禁止（RC4/3DES/NULL）
- KMS/鍵アクセス制御
  - 環境別CMK
  - 365日ローテーション
  - 30日前に期限アラート

## シークレット
- 保管場所: Secret Manager
- 参照方法: サービスアカウントのみ許可
- ローテーション: 90日
- スキャン運用: 検知時は24時間以内にローテーション

## 脆弱性対応SLA
- Critical: 7日以内
- High: 30日以内
- Medium: 90日以内
- Low: 180日以内

## WAF/DDoS
- ルールセット: OWASP CRS
- レート制限: 100 rps/IP（5分平均）
- 例外手順: 申請→一時許可（最大24h）
- 検証方法: 変更前後で誤検知率を計測
