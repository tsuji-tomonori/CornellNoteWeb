# バックアップ/復元/DR（詳細）

## 対応NFR
- NFR-AV-005, AV-006, AV-009
- NFR-PRC-005（リージョン制約）
- NFR-COST-005

## バックアップ設定
- DB
  - PITR: 15分間隔（連続アーカイブログ）
  - フルバックアップ: 毎日 02:00 JST
  - 週次フル: 毎週日曜 03:00 JST（保持 12ヶ月）
  - 保持期間: 90日（短期）+ 12ヶ月（週次）
  - バックアップ成功率: 月次 99%以上
- Object Storage
  - バージョニング: 有効
  - ライフサイクル
    - 30日でIA
    - 365日でアーカイブ
    - 730日で削除
- 暗号化: すべてKMS管理鍵で暗号化

## 復元手順（四半期訓練）
- 手順
  1. DBを指定時点に復元（PITR）
  2. アプリ疎通確認（ヘルスチェック/主要API）
  3. 整合性チェック（件数/参照/主要集計）
- 検証観点
  - 件数一致（差分0.1%以内）
  - 参照整合性エラー 0件
  - 主要集計（DATA-004連携）
- 記録
  - 実施日時/手順/結果/所要時間

## DR手順（年2回テスト）
- 構成
  - セカンダリリージョンにスタンバイ環境
  - データ複製: 15分以内で同期
- 切替条件
  - 主要リージョン障害 30分以上
  - SLO違反が継続（可用性 99.0%以下）
- 切替手順
  1. DNS/ロードバランサ切替
  2. アプリ起動/疎通
  3. 監視切替
- 戻し手順
  - 再同期後に段階切戻し
- RTO/RPO
  - RTO <= 4h / RPO <= 15m（訓練で実測）
- データ所在地制約
  - 制約地域は同地域内DRを優先、不可時は暗号化+法務承認
