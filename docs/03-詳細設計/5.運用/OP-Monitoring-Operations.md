# 監視運用

## 目的
SLI/SLOに基づきサービス品質を継続監視し、障害の早期検知と迅速復旧を実現する。

## 対応NFR
- NFR-OPS-001〜006, OPS-008, OPS-009
- NFR-AV-001
- NFR-PS-001, PS-003, PS-004

## 監視対象
- RED/USEメトリクス（詳細: DD-Observability-MetricsAlerts.md）
- ログ/監査ログ（詳細: DD-Observability-Logging.md）
- ヘルスチェック（Liveness/Readiness）
- 主要ユースケースの合成監視

## アラート運用
- 重大: Pager通知、即時一次対応（15分以内）
- 中: Slack通知、30分以内に調査開始
- 低: Email通知、24時間以内に分析
- 月次で通知テストを実施（OPS-003）

## SLO/エラーバジェット運用
- 月次SLOレポートを作成し、エラーバジェット消費を可視化
- 消費率が30%に達した場合は変更凍結/品質改善を優先
- 消費率が100%到達でリリース停止と緊急改善計画を発動

## ログ検索/トレース運用
- traceIdによる障害調査を標準化
- 監査ログの検索はセキュリティ担当のみ実施
- PIIマスキング方針を遵守（PRC-002）

## メトリクス保持
- 30日以上保持し、容量計画/性能改善に利用
