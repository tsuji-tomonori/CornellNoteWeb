# 権限/シークレット運用

## 目的
最小権限と監査性を確保し、権限乱用や情報漏えいを防ぐ。

## 対応NFR
- NFR-SEC-001〜006, SEC-008, SEC-009, SEC-010
- NFR-PRC-004

## 権限管理
- IAMロールはDD-Security-Controls.mdの定義に準拠
- 特権操作はMFA必須、条件付きアクセスを適用
- 例外権限は期限付き（最大7日）とし、事後レビュー

## 権限レビュー
- 半期ごとに権限棚卸し
- 不要権限は即時削除

## シークレット管理
- シークレットはSecret Managerで一元管理
- リポジトリ混入はCIでブロック
- ローテーション: 90日ごと
- インシデント時は24時間以内に緊急ローテーション

## 監査ログ
- 認証/権限変更/シークレット参照は監査対象
- 監査ログはWORM相当で365日以上保管
