# 障害/問題管理

## 目的
障害の影響を最小化し、再発防止までを一貫して管理する。

## 対応NFR
- NFR-OPS-003, OPS-004
- NFR-AV-007, AV-009
- NFR-DEV-004
- NFR-SEC-008

## 障害対応フロー
1. 検知（監視/問い合わせ）
2. 一次対応（ランブックに従い影響を抑制）
3. エスカレーション（15分/30分ルール）
4. 復旧（ロールバック/フェイルオーバ等）
5. 原因分析（RCA）
6. 再発防止策の実施

## 重大度定義（例）
- SEV1: 全面停止/SLO重大違反
- SEV2: 主要機能の部分停止
- SEV3: 限定機能の不具合

## ポストモーテム
- SEV1/2は100%実施
- docs/04-運用/postmortems/ に保存
- 再発防止タスクはチケット化して追跡

## 連絡/報告
- 影響・暫定対応・復旧見込みを30分以内に共有
- 利用者向け通知はリリースノート/ステータスページで実施

## ランブック連携
- 具体手順は docs/04-運用/runbooks/ に定義
- 対象: API重大障害/DBフェイルオーバ/バックアップ復元/セキュリティ初動
