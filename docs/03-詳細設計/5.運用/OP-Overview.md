# 運用概要

## 目的
安定稼働・セキュリティ・法令遵守を満たしつつ、SLO/SLAを継続的に達成するための運用方針を明確化する。

## 対応NFR
- NFR-AV-001〜010
- NFR-OPS-001〜009
- NFR-DEV-002〜008
- NFR-SEC-001〜012
- NFR-PRC-001〜008
- NFR-COST-001〜005
- NFR-UX-001〜026
- NFR-DATA-001〜005

## 運用原則
- 予防: 監視/自動化/品質ゲートで障害の発生確率を下げる
- 早期検知: SLI/SLOとアラートで影響を最小化
- 迅速復旧: ランブック/訓練/ロールバックでMTTRを短縮
- 監査性: 変更/アクセス/障害の記録を保管し提示できる

## 体制と責任分界
- プロダクト責任: 仕様/リリース/障害対応の最終責任
- 運用担当: 監視/障害一次対応/変更実施
- セキュリティ担当: 権限/監査/インシデント対応
- 開発担当: 不具合修正/再発防止/改善

## 主要指標
- 可用性SLO: 99.9%（月次）
- 主要API p95: 300ms以下（PS-001）
- エラーバジェット: OPS-008に準拠
- 復旧目標: RTO <= 4h / RPO <= 15m
